# Патч 12 — модуль «КП» (генерація комерційних пропозицій)

## Що нового

- Пункт меню **«КП»** (лише керівник відділу) з трьома вкладками:
  - **Складання КП** — тип обладнання (лазер / ЧПК / універсальні / інше), назва
    верстата, ціна/валюта/термін постачання (необов'язково), зона завантаження
    файлів з drag&drop, вибором і вставкою з буфера (Ctrl+V), кнопка «Згенерувати».
  - **Бібліотека** — усі згенеровані КП: фільтр за типом, пошук за назвою, дата,
    автор, скачування, видалення.
  - **Шаблони** — редагування системних інструкцій ШІ за напрямами (з кнопкою
    скидання до стандартних). База — інструкція «Генерація КП Стан Комплект» v2.
- Генерація: PDF і фото йдуть у Anthropic API нативно, DOCX — через mammoth,
  XLSX — через ExcelJS. Результат — фірмовий DOCX (Montserrat, #009C4B/#F36E33,
  структура: обкладинка → про компанію → про верстат → характеристики →
  комплектація + опції → комерційна частина, правило ЧПК ↔ УЦІ).
- Файли КП зберігаються на тому ж Railway Volume у підпапці `kp/`
  (`UPLOADS_DIR`), записи — у нових таблицях `KpTemplate` / `KpDocument`
  (міграція `20260723090000_kp_module`, накотиться автоматично при деплої).

## Кроки після розпакування

1. **Фірмові ассети (важливо для повного оформлення).** Покладіть у `assets/kp/`
   чотири PNG з Project knowledge проєкту «КП»: `SK_cover_base.png`,
   `SK_footer.png`, `SK_company_top.png`, `SK_company_bottom.png`
   (див. `assets/kp/README.md`). Без них генерація працює у спрощеному вигляді
   з попередженнями.
2. **Змінні середовища Railway.** `ANTHROPIC_API_KEY` вже використовується для
   актів — додаткових змінних не потрібно. За бажанням: `ANTHROPIC_KP_MODEL`
   (модель саме для КП, за замовчуванням — як `ANTHROPIC_MODEL`).
3. **Деплой як зазвичай:**

   ```powershell
   cd C:\Projects\sk-service
   git add -A
   git commit -m "Patch 12: KP module"
   git push
   ```

4. Після деплою відкрийте **КП → Складання КП**, зробіть тестову генерацію
   (будь-який PDF з ТХ + фото). Якщо назва верстата на обкладинці не потрапляє
   в сіру смугу — підженіть `COVER_TITLE.y` у `src/lib/kp/constants.ts`
   (1 см ≈ 567 DXA) і задеплойте ще раз.

## Технічні нотатки

- Нові залежності: `docx`, `mammoth`, `image-size` (у package.json/lock,
  встановляться на Railway автоматично).
- Генерація займає 30–120 с залежно від обсягу PDF; інтерфейс показує фази
  та таймер, з'єднання тримається (Railway це дозволяє).
- Ліміти: до 10 файлів, до 25 МБ кожен. Формат .doc (старий Word) не
  підтримується — конвертуйте в .docx або PDF.
- Фото: перше завантажене зображення — на обкладинку та великим кадром у
  «Про верстат», наступні (до трьох) — рядом деталей. WebP/HEIC у DOCX не
  вставляються (лише JPG/PNG/GIF) — для аналізу ШІ вони при цьому
  використовуються.
